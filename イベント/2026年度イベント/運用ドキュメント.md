## 1. システム構成

### 全体像
Webカレンダー（GitHub Pages）を中心に、Googleスプレッドシート・Google Drive・Google Apps Script（GAS）を連携させた仕組み。

- **表示**: GitHub Pages（静的HTML）
- **データ保存**: Googleスプレッドシート（予約・実績）
- **ファイル保存**: Google Drive（プレゼン資料）
- **API**: GAS（WebカレンダーとGoogle系サービスの橋渡し）
- **ローカル管理**: Googleドライブデスクトップアプリで同期

### データの流れ
```
[Webカレンダー] --fetch/POST--> [GAS] --appendRow/getValues--> [スプレッドシート]
[Googleフォーム] --自動--> [同じスプレッドシートの別シート]
[GAS doGet] --> 両シートを合算して返す --> [Webカレンダーに表示]
```

### プッシュ方法
```
cd "北総フォルダ２"
git add ファイル名
git commit -m "メッセージ"
git push origin main
```
→ 数分後にGitHub Pagesに反映

- リポジトリは**公開（public）**でないとGitHub Pagesが動かない（無料プラン）
- 非公開にするとカレンダーが見れなくなる
- コード自体は見られるが、スプレッドシートのデータやGoogle Driveファイルは直接アクセスされない

## 5. GAS更新手順（毎回同じ）

1. コードファイル（.txt）をテキストエディットで開く
2. **テキストエディットが古い場合があるので、一度完全に閉じてから開き直す**
3. Cmd+A → Cmd+C でコピー
4. スプレッドシート → 拡張機能 → Apps Script でGASエディタを開く
   （アップロード用は script.google.com → プロジェクト一覧から）
5. GASエディタで Cmd+A → Cmd+V → Cmd+S
6. 「デプロイ」→「新しいデプロイ」→ ウェブアプリ → 実行:自分 → アクセス:全員 → デプロイ
7. 新URLをindex.htmlに反映
8. GitHubにプッシュ

## 6. スプレッドシート共有の仕組み

### 方針
- ローカルのGoogleドライブデスクトップ → クラウドに同期が基本
- カレンダーは自分が作成、Googleフォームは別の担当者が作成
- 自分のスプレッドシートをフォーム担当者に「編集者」で共有
- フォームの回答先を自分のスプレッドシートに設定してもらう

### 共有済みスプレッドシート
- サークル予約（`1UOmKA8c2W84lKa1b2ol5S4KHf1caD8df2ioCecg1xvg`）
- セミナー予約（`1Q5jAK7SKxbrzfuvGNtH4hDoKM0ar2aPodf9CVtkzjqI`）
- サークル実績報告（`1KsbJuCS1aSYFvqaEzHHz_7vgnwelz1Sv62vNoW9aAms`）
- セミナー実績報告（`1OWfwdbjz5tnK5zro_7XTrvY9F21CAgGZS2P_fM7vGEo`）

### シート分離の理由
Googleフォームが接続されたシート（「フォームの回答 1」等）にGASで書き込むと、フォーム側がデータを上書き・競合する問題が発生した。そのため：
- **Webからの入力**: `Web予約` / `Web実績` シートに書き込み
- **Googleフォームからの入力**: `フォームの回答 X` シートに書き込み
- **表示時**: 両方を合算して返す

## 9. 既知の問題と対処

### テキストエディットのキャッシュ
GASコードを更新した後、テキストエディットが古い内容を表示することがある。
→ テキストエディットを `Cmd+Q` で完全終了してから開き直す。

### イベント名マッチングの仕組み
WebカレンダーとGoogleフォームでイベント名の形式が異なる。
- Web: `2026-04-23 富里 タイトル`
- フォーム: `4/23(木) 13:00 富里 サークル名 タイトル DX`
→ 日付（月/日）+ 会場の一致で照合している。部分一致ではなく厳密一致。

### GitHub Pagesが止まった場合
リポジトリの非公開/公開切り替えでPagesが無効になることがある。
→ `gh api repos/.../pages -X POST` で再有効化する。
