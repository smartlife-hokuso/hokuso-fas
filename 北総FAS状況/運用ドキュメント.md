## 3. 機能一覧

### トップページ（index.html）
- 統計アイコン: プランナー数、拠点数、総担当軒数、総AO生数、総DX数
- 累計アセットTOP10ランキング
- プランナー名検索
- 地域別プランナーカード（軒数/人数/AO/DX表示）
- 変更ログ閲覧リンク
- データ取得: JSONP方式、キャッシュバスター付き（`Date.now()`）

### 個人ページ（プランナー/*/index.html）
7タブ構成:
1. **FAS状況** — 担当軒数/人数、聴講生、会員数等（累計）
2. **WIN状況** — WIN10/11切替状況
3. **アセット実現（当月）** — VC、AO生、専門生、DX等13項目
4. **イベント** — 来館、参加確定、参加予定
5. **アポイント（当月）** — コール数、ラウンド数、確定軒数/人数
6. **人材獲得** — リファラル、FAS-Z
7. **履歴（単日）** — 日付別変更履歴、当月アセット合計、当日アセット件数

### 編集・保存
- 「編集する」ボタンで編集モード → 数値入力 → 「保存する」でスプレッドシートに保存
- 保存時に変更ログシートに自動記録（日時・名前・項目・変更前・変更後）

### 履歴表示ルール
- 初回の累計データ（oldVal = newVal）は `0 → 値` と表示
- 通常の変更はそのまま表示
- 同日に複数回入力した場合は最新のみ表示
- 0→0の項目は非表示
- 当日アセット件数 = アセット13項目の値の差分合計（初回は「累計初期値」と表示）

## 4. GAS API仕様

**ベースURL:** `https://script.google.com/macros/s/{デプロイID}/exec`

| action | 説明 | パラメータ |
|--------|------|-----------|
| （なし） | プランナーデータ読み取り | `name=名前` |
| `save` | データ保存 | `name=名前&data=JSON` |
| `totals` | 全体統計（トップページ用） | なし |
| `log` | 変更ログ全件（管理者用） | `pw=hokuso2026` |
| `mylog` | 個人変更ログ | `name=名前` |

- JSONP対応: `callback=関数名` パラメータで対応
- 日付はgetDisplayValues()で読み取り（タイムゾーン二重変換防止）

### 計算定義
- **AO生数** = 内一般聴講生軒数 + 内専門聴講生軒数 + 内DX会員軒数
- **DX数** = 内DX会員軒数
- **アセット合計** = VC + 内AO生 + 内専門生 + 内ビジネス + 内DXエントリー + 内DXレギュラー + CEX + 従業員オプション + WEB + GoogleWokSpace + GenSpark + 光/SIM/モバイルルータ + 我が社

## 5. 運用手順

### GASコード更新手順
1. `copy_code.html` をブラウザで開く
2. 「コードをコピー」ボタンを押す
3. スプレッドシート → 拡張機能 → Apps Script を開く
4. 既存コードを全選択（Cmd+A）→ 削除 → 貼り付け（Cmd+V）→ 保存（Cmd+S）
5. 「デプロイ」→「新しいデプロイ」→ ウェブアプリ → アクセス: 全員 → デプロイ
6. 新しいURLを取得
7. 全HTMLファイルのURL一括置換:
   ```bash
   cd 北総FAS状況
   find . -name "*.html" -exec sed -i '' 's|旧URL|新URL|g' {} +
   ```
8. Git コミット & プッシュ:
   ```bash
   git add 北総FAS状況/
   git commit -m "GAS新デプロイURL更新"
   git push fas main
   ```

### プランナー追加・削除手順
1. `プランナー一覧.xlsx` を更新
2. `planners.json` を再生成（Pythonスクリプト）
3. 新規の場合: プランナーフォルダ作成（data.json + index.html）
4. `index.html` のHTML部分を更新（拠点セクションに追加/削除、人数更新）
5. スプレッドシートの名前行にも追加/削除
6. Git コミット & プッシュ（`git push fas main`）

### プランナー所属変更手順
1. `planners.json` の cd と base を変更
2. `プランナー/*/data.json` の cd と base を変更
3. `index.html` で旧拠点から削除、新拠点に追加、人数更新
4. スプレッドシートの2行目（拠点名）を変更
5. Git コミット & プッシュ

### 月初リセット手順（毎月1日に実施）
当月項目（アセット・アポイント）を0リセットし、前月分は別シートへアーカイブする。

**当月項目（リセット対象 計17項目）:**
- アセット13項目: VC、内AO生、内専門生、内ビジネス、内DXエントリー、内DXレギュラー、CEX、従業員オプション、WEB、GoogleWokSpace、GenSpark、光/SIM/モバイルルータ、我が社
- アポイント4項目: アポイントコール数、アポイントラウンド数、アポイント（確定）軒数、アポイント（確定）人数

**累計項目（継続）:**
- FAS状況全項目（担当軒数・人数、聴講生、会員数、引継、総デバイス数、他DLPのDX担当軒数等）
- WIN状況（WIN10使用中、11切替済、不要、未確認）
- イベント（イベント来館、イベント参加確定、イベント参加予定）
- 人材獲得（リファラル、FAS-Z、FAS-Z（学生））

**⚠️ 重要: 実行タイミング**
- **月末締切後・翌月1日朝の「誰も入力していない状態」で実行する**
- 月途中で実行すると、既に入力済みの当月分が0で上書きされる
- 万一誤って実行した場合は `restoreFromArchive` 関数で取り消し可能（後述）

**手順:**
1. スプレッドシート → 拡張機能 → Apps Script を開く
2. `archiveAndResetMonth` 関数の冒頭にある `archiveName` を当月の前月名（例: `"2026年4月"`）に書き換え
3. 関数選択ドロップダウンで `archiveAndResetMonth` を選び、▶ で実行
4. 実行ログに `アーカイブし、17項目を0リセットしました` が出れば成功
5. スプレッドシートを開き、新規シート（例: `2026年4月`）が末尾にできていることと、現シート（`SL富里のコピー`）の当月17項目が0になっていることを確認
6. 公開ページ（トップページ）のTOP10ランキングが当月のアセット実績に切り替わる（リセット直後は全員0）

### apo.ozzio.info 自動同期（GAS完結方式・2026-05-07刷新）
apo.ozzio.info 公式外部API → FAS スプレッドシートへ1時間ごと自動同期。  
GASのUrlFetchAppとScriptAppの時間トリガーだけで完結。**Mac/Python/launchd不要**。

**外部API仕様（apo.ozzio.info 提供）:**
```
GET /api/external/monthly?year=YYYY&month=M    # 月次集計（同期で使用）
GET /api/external/weekly?start=YYYY-MM-DD&end=YYYY-MM-DD
GET /api/external/daily?date=YYYY-MM-DD

認証: X-API-Key ヘッダー（または ?api_key= クエリ）
レスポンス: {"success": true, "data": [{planner_name, total_outbound_*, total_inbound_*}]}
```

**GAS側の関数:**
- `setApoApiKey(key)` — APIキーをScriptPropertiesに保存（暗号化、コードに直書きしない）
- `fetchApoMonthlyAndSync()` — 外部API呼出→`syncFromApo`で反映（時間トリガーから呼ばれる）
- `setupAutoApoSync()` — 1時間ごとの時間トリガー設定（1回だけ実行）
- `removeAutoApoSync()` — 自動同期停止

**マッピング（apo 24名 → FAS 23名）:**
GAS_code.js の `APO_TO_FAS_NAME` 定数で管理。`石井（俊）` のみ対応プランナーがFAS側で削除済みのためスキップ。

**FAS 3項目への変換ルール（GAS の `transformApoRowToFasValues`）:**
- アポイントコール数 = `total_outbound_calls + total_inbound_calls`
- アポイント（確定）軒数 = `total_outbound_visit_promises + total_inbound_visit_promises`
- アポイント（確定）人数 = `total_outbound_expected_visitors + total_inbound_expected_visitors`
- アポイントラウンド数 = apo側に対応項目なし → 同期しない（手入力のまま）

**初回セットアップ手順:**
1. GASエディタで `setApoApiKey` 関数のデフォルト引数（`PUT_YOUR_API_KEY_HERE` の場所）に発行されたAPIキーを書き込む、または `setApoApiKey("実キー")` を別関数として呼ぶ
2. `setApoApiKey` を ▶ 実行 → 「APIキーを保存しました」と返る
3. `setupAutoApoSync` を ▶ 実行 → 1時間トリガー設定完了
4. `fetchApoMonthlyAndSync` を ▶ 実行で動作テスト

**同期方式: 差分加算（2026-05-06より）**
apo値で **上書きせず** 差分（今回apo - 前回apo）を現在のFAS値に加算する。
- 「apo同期状態」シート: プランナー名 / 項目名 / 前回apo値 / 最終同期日時
- これによりプランナーがFASで手入力した値が次回同期で消えなくなる
- 月初リセット時（`monthlyResetTrigger`）は「apo同期状態」も自動クリア

**ブートストラップ動作（初回・新規プランナー）:**
状態シートに記録がない場合：
- FAS=0 かつ apo>0 → FAS に apo値を初期投入
- FAS>0 → FAS は触らず状態のみ記録（既に同期済みと仮定）

**競合検知:**
同一日に同じプランナー・同じ項目が「apo」と「manual」両方から更新されると、`競合ログ`シートに自動記録される。FAS Webトップページに `⚠️ apoと手入力の両方に入力された日があります` バナーが表示される（クリックで詳細展開）。

**実行ログ:**
「自動実行ログ」シートに毎回の同期結果（成否・反映件数・ブートストラップ件数・競合件数）が記録される。

### 自動月次リセット（推奨）
GASの時間トリガーで毎月1日 03:00 JST に自動実行できる。

**1回だけセットアップ:**
1. GASエディタで `setupAutoMonthlyReset` を選択 → ▶ で実行
2. 「毎月1日 03:00 (JST) に...設定しました」が返れば完了
3. 以降は何もしなくてよい。自動で `monthlyResetTrigger` が走る

**自動実行の中身:**
1. 前月分を `recomputeMonthFromLogs` で純粋な月次値に再計算
2. 前月名（例: `2026年5月`）でシートをアーカイブ
3. 現シートの当月17項目を0リセット
4. 「自動実行ログ」シートに実行日時・対象月・結果を記録

**注意事項:**
- 月末23:00〜翌月04:00の間は入力しないこと（トリガー前後の入力は前月分扱いになる可能性）
- トリガー実行に失敗した場合は「自動実行ログ」シートにエラーが記録される
- 自動運用を止めたい場合は `removeAutoMonthlyReset` を実行

### リセット取り消し手順（誤実行時）
誤って当月途中でリセットした場合、アーカイブシートから当月17項目を復元できる。

1. GASエディタで `restoreFromArchive` 関数の `archiveName` を復元元のシート名に設定（直前のアーカイブと一致させる）
2. ▶ で実行
3. 実行ログに `アーカイブから 17 項目を復元しました` が出れば成功
4. 復元後はアーカイブシートに「リセット時点の値」が残ったまま。次回正規の月初リセットを実施する前に、不要なら手動で削除しても良い
